# 01 — ТЗ: Дизайнер

Прочитай 00-OVERVIEW.md. Твоя задача — визуальная система и прототипы экранов,
которые разработчик перенесёт в Next.js без додумывания.

## Художественное направление

Продукт для владельца салона/клиники, который только что получил гневный отзыв и нервничает.
Интерфейс должен снижать тревогу и давать ощущение «ситуация под контролем профессионала».
Метафора: операционный пульт репутации. Серьёзно, спокойно, дорого. НЕ игрушечно,
НЕ стартаперский неон ради неона.

Требования к направлению:

- Тёмная тема как основная (light — вне scope). Фон — глубокий графит с тёплым подтоном,
  не чисто-чёрный.
- Один акцентный цвет для AI-действий (генерация, готовность) + семантическая шкала
  severity 1–5 (от спокойного к тревожному) — severity-индикатор это фирменный элемент
  продукта, придумай его форму (не просто цветной бейдж).
- Типографика: характерный display-шрифт для заголовков и цифр + нейтральный текстовый.
  Обязательна поддержка кириллицы (проверь каждое начертание!). Подбери пару осознанно,
  не Inter+Inter.
- Один «signature»-элемент на весь продукт: анимированный пайплайн генерации
  «Отзыв → Анализ → Тональность → Пакет» с последовательным появлением карточек.
  Это главный wow-момент демо — потрать смелость здесь, остальное держи тихим.
- Уважай prefers-reduced-motion: статичная версия пайплайна.

## Артефакты (положи в docs/design/)

1. `tokens.css` — CSS-переменные: палитра (4–6 именованных hex), шкала severity,
   радиусы, тени, spacing-шкала, типографическая шкала (размер/вес/межстрочный),
   длительности и easing-функции анимаций.
2. `ui-kit.html` — кнопки (primary/secondary/ghost, состояния hover/focus/disabled/loading),
   инпуты и textarea, табы, карточка, тост, бейджи источников (Яндекс/2ГИС/Ozon/WB),
   severity-индикатор, скелетоны загрузки.
3. Прототипы экранов, самодостаточные HTML+CSS (без JS-фреймворков, ванильный JS для
   демонстрации анимаций допустим):
   - `screen-auth.html` — логин/регистрация
   - `screen-onboarding.html` — 3 шага: компания и ниша → тон бренда → примеры текстов
   - `screen-generate.html` — главный экран: textarea отзыва, выбор источника и рейтинга,
     кнопка генерации, ТРИ состояния: пустое / анимация пайплайна / результат с 4 карточками
   - `screen-history.html` — список отзывов с фильтрами (источник, категория, severity, дата)
   - `screen-settings.html` — настройки компании и тона
4. `MOTION.md` — раскадровка анимации пайплайна: тайминги, порядок появления карточек,
   что происходит при ошибке генерации.

## Карточки результата — обязательные элементы

- Карточка «Публичный ответ»: 3 таба тона, текст, кнопка «Скопировать» с подтверждением,
  пометка площадки («адаптировано под Яндекс.Карты»).
- Карточка «Внутренняя задача»: структура что/причина/проверить/кому, кнопка копирования.
- Карточка «Классификация»: категория, severity-индикатор, блок повторяемости
  («3-й похожий отзыв за месяц» + ссылки на прошлые), флаг «похоже на заказной отзыв»
  с объяснением — спроектируй так, чтобы флаг не выглядел обвинением.
- Карточка «Возврат клиента»: сообщение + рекомендация компенсации.

## Тексты интерфейса

Пиши реальный русский UX-копирайт, не lorem ipsum. Голос: спокойный профессионал,
глаголы действия, sentence case, без восклицаний. Ошибки объясняют, что делать дальше.
Пустые состояния приглашают к действию. Кнопка называет результат:
«Сгенерировать пакет реакции», не «Отправить».

## Критерии приёмки

- Все экраны корректны на 360px и 1440px.
- Контраст текста ≥ 4.5:1, видимый focus-ring на всех интерактивных элементах.
- Прототипы открываются в браузере без сборки.
- Перед сдачей: самокритика — если палитра/шрифты выглядят как «дефолтный AI-дашборд
  с фиолетовым неоном», переработай. Зафиксируй 3 осознанных отличия в docs/design/NOTES.md.
