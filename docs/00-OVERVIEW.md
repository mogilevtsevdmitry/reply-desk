# ReplyDesk — общий контекст проекта (читают ВСЕ агенты)

## Продукт

ReplyDesk — AI-сервис для локального бизнеса (салоны, стоматологии, рестораны, автосервисы, фитнес).
Пользователь вставляет текст отзыва клиента (Яндекс.Карты, 2ГИС, Ozon, WB) и получает за ~8 секунд
«пакет реакции» из четырёх блоков:

1. **Публичный ответ** — 3 варианта тона (мягкий / нейтральный / уверенный), с учётом правил площадки.
2. **Внутренняя задача** — что произошло, вероятная причина, что проверить, кому назначить (роль).
3. **Классификация** — категория проблемы (сервис / качество / персонал / цена / ожидание),
   severity 1–5, признак повторяемости (поиск похожих прошлых отзывов через pg_trgm
   с подтверждением LLM),
   флаг подозрения на заказной отзыв.
4. **Возврат клиента** — личное сообщение клиенту + рекомендация по компенсации.

## Scope MVP (строго)

ВХОДИТ: регистрация/логин (email+пароль), онбординг компании (название, ниша, тон бренда,
примеры текстов), ручная вставка отзыва, генерация пакета с live-статусами (SSE),
история отзывов с фильтрами, лимиты генераций по тарифу (Free: 10/мес), копирование текстов.

НЕ ВХОДИТ: автосбор отзывов с площадок, автопостинг ответов, Telegram-бот, оплата,
мультиязычность, командные роли (в MVP один пользователь = одна компания, схема данных
закладывается на расширение).

## Стек (зафиксирован, не менять)

- Монорепо: pnpm workspaces + Turborepo
- Frontend: Next.js 15 (App Router), TypeScript, Tailwind CSS 4
- Backend: NestJS 11, Prisma 6, PostgreSQL 16 (+ pg_trgm), Redis 7 + BullMQ
- AI: провайдер-агностичный LLM-адаптер (интерфейс `LlmProvider`), реализация AnthropicProvider
  (модель из env), structured output по JSON-схеме. Anthropic API — единственный внешний
  AI-сервис; embeddings не используются, похожие отзывы ищутся через pg_trgm (см. DECISIONS ADR-001)
- Realtime: SSE (Server-Sent Events) для статусов генерации
- Авторизация: JWT (access 15m + refresh 30d, refresh в httpOnly cookie)

## Доменный словарь

- **Company** — бизнес клиента (тенант). **Review** — вставленный отзыв.
- **Generation** — результат AI-обработки одного отзыва (4 блока).
- **ToneOfVoice** — настройки тона компании + примеры текстов.
- **Niche skill** — md-файл со спецификой ниши для промпта (prompts/niches/*.md).

## Definition of Done проекта

`docker compose up` поднимает всё с нуля; пользователь регистрируется, проходит онбординг,
вставляет отзыв, видит анимированную генерацию и 4 карточки, копирует ответ; история
сохраняется; лимит Free-тарифа срабатывает; тесты зелёные; критичные находки безопасника закрыты.
