# 06 — ТЗ: Контент-рерайтер

Прочитай 00-OVERVIEW.md, раздел «Тексты интерфейса» в 01-DESIGN.md и все прототипы
в docs/design/*.html (черновой копирайт уже там). Твоя задача — единый выверенный
копидек всех текстов приложения, который разработчик перенесёт в код без додумывания.
Прототипы НЕ редактируй — копидек становится источником истины при расхождениях.

## Голос продукта

Пользователь — владелец салона/клиники, который только что получил гневный отзыв
и нервничает. Голос: спокойный профессионал, который берёт ситуацию под контроль.
Инфостиль: глаголы действия, конкретика вместо абстракций, sentence case,
без восклицаний, без канцелярита и без панибратства. Кнопка называет результат
(«Сгенерировать пакет реакции», не «Отправить»). Ошибки объясняют, что делать дальше.
Пустые состояния приглашают к действию, а не констатируют пустоту.

## Артефакт: docs/content/COPY.md

Один файл, структурированный по экранам и системным блокам. Для каждой строки —
стабильный ключ (kebab-case, станет ключом в коде), текст, при необходимости —
комментарий по контексту/ограничению длины.

### Покрыть обязательно

1. **Auth** — вход, регистрация, плейсхолдеры, ошибки полей (email занят, слабый
   пароль ≥ 8 символов, неверные данные — одно сообщение для «нет юзера» и «не тот
   пароль»), ссылки переключения форм.
2. **Онбординг, 3 шага** — заголовки/подписи шагов, поля (название, ниша — подписи
   всех 7 значений enum Niche), выбор тона (3 значения с человеческими описаниями),
   примеры текстов (подсказка: до 3 примеров по 1000 символов), кнопки навигации.
3. **Главный экран генерации** — пустое состояние, плейсхолдер textarea, счётчик
   символов (до 4000), подписи источников (5 значений ReviewSource), рейтинг,
   кнопка генерации, подписи 4 узлов пайплайна (Отзыв → Анализ → Тональность → Пакет),
   статусные реплики процесса.
4. **4 карточки результата** — заголовки, табы тона (мягкий/нейтральный/уверенный),
   пометка площадки («адаптировано под Яндекс.Карты» × 5 источников), структура
   внутренней задачи (что/причина/проверить/кому + подписи ролей), блок повторяемости
   («N-й похожий отзыв за месяц»), флаг заказного отзыва — формулировка-предположение
   без обвинения, карточка возврата клиента (сообщение, рекомендация компенсации).
5. **Ошибки и лимиты** — FAILED-генерация (+ «Лимит не потрачен», кнопка «Повторить
   генерацию»), 402-экран апгрейда (заглушка без оплаты: что такое тарифы START/BUSINESS,
   CTA-заглушка), счётчик лимита в сайдбаре («7 из 10»), сетевые ошибки, 429 rate limit.
6. **История** — заголовок, фильтры (источник, категория — подписи 6 значений enum
   Category, severity, даты), пустые состояния (нет отзывов вообще / ничего не найдено
   по фильтрам), подписи severity 1–5 (словесные метки уровней).
7. **Настройки** — секции компании и тона, кнопки сохранения, тост успеха.
8. **Системное** — тосты (скопировано, сохранено, ошибка), подтверждения, диалог
   выхода, title страниц, мета-описание лендинга не нужно (вне scope).

## Критерии приёмки

- Ни одной строки-заглушки: каждый текст готов к продакшену как есть.
- Терминология едина по всему копидеку (один глоссарий: «пакет реакции», «генерация»,
  «отзыв», «площадка» — не синонимить).
- Ошибки: каждая отвечает на «что случилось» и «что делать», без кодов и стектрейсов.
- Severity 1–5 имеют словесные метки, согласованные с дизайном (см. ui-kit.html).
- Расхождения с черновым копирайтом прототипов — отдельным списком в конце COPY.md
  (что заменено и почему), чтобы дизайнер/разработчик видели дельту.
- Если ТЗ молчит — простейшее решение + ADR в docs/DECISIONS.md.
