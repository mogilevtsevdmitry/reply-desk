# COPY.md — копидек ReplyDesk

Источник истины для всех текстов интерфейса. При расхождении с прототипами
docs/design/*.html действует этот файл (см. список расхождений в конце).

**Конвенции.**
- Ключ — стабильный kebab-case, переносится в код как есть.
- Плейсхолдеры — в фигурных скобках: `{n}`, `{limit}`, `{month}`. Подстановка на фронте.
- Русские плюралы: где нужны формы слова, они даны явно в комментарии
  (формат one/few/many для Intl.PluralRules: `ru`).
- Голос: спокойный профессионал, инфостиль, sentence case, без восклицаний.
  Кнопка называет результат. Ошибка отвечает «что случилось» и «что делать».

**Глоссарий (не синонимить).**
- **пакет реакции** — результат генерации из 4 блоков (не «ответ AI», не «результат анализа»)
- **генерация** — единица расхода лимита (не «запрос», не «обработка»)
- **отзыв** — текст клиента (не «сообщение», не «жалоба»)
- **площадка** — источник отзыва (не «платформа», не «сервис»)
- **пульт** — брендовая метафора приложения, используется дозированно (логин, финал онбординга)

## Расхождения с черновым копирайтом прототипов

Прототипы не редактировались; при переносе в код действует этот копидек.

1. **Добавлена ниша «Медицинская клиника»** (`niche-medical`). В enum Niche семь
   значений, в прототипах онбординга и настроек — только шесть чипов/опций.
   Без неё нишевый промпт medical.md недостижим из UI. (ADR-009)
2. **Добавлена площадка «Другая площадка»** (`source-other`) в выбор источника и
   фильтр истории + пометка карточки ответа `card-reply-note-other` «без привязки
   к площадке». В enum ReviewSource пять значений, в прототипах — четыре. (ADR-009)
3. **Добавлена категория «Другое»** (`category-other`) в фильтр истории и карточку
   классификации. В enum Category шесть значений, в фильтре прототипа — пять. (ADR-009)
4. **Чип категории «Качество услуги» → «Качество»**. Единая терминология: метка
   категории одинакова в карточке классификации и фильтре истории; «услуги» не
   подходит товарным отзывам Ozon/WB.
5. **Шаг 3 онбординга и поле примеров в настройках: одна textarea → три отдельных
   поля** с счётчиком «{n} / 1000». Контракт хранит examples: string[] (до 3 × 1000);
   разбор одной textarea по пустым строкам — неочевидная магия и невалидируемая
   длина. (ADR-010)
6. **Флаг заказного отзыва, позитивное состояние** — в прототипе показано только
   «чистое» состояние. Добавлен заголовок-предположение «Похоже на заказной отзыв»
   и переформулирован дисклеймер: «Оценка алгоритма, а не вывод» →
   «Это предположение алгоритма, а не вывод» — слово «оценка» в этом экране занято
   оценкой клиента (звёзды), «предположение» точнее снимает обвинительность.
7. **Добавлены тексты, которых в прототипах нет совсем:** все ошибки полей auth,
   ошибки валидации отзыва (пустой/длиннее 4000), сетевые ошибки, 429,
   обрыв SSE, истёкшая сессия, экран 402 (ADR-012), пустые состояния истории,
   диалог выхода, состояние «лимит на нуле» у формы генерации, состояние
   «похожих отзывов не нашли».
8. **Тост ошибки ui-kit** «Не удалось сохранить — повторите попытку» закреплён как
   `toast-save-error`; универсальная серверная ошибка получила отдельный текст
   `error-server` с ответом «что делать» (попробовать через минуту).
9. **Подпись лимита в сайдбаре** параметризована: «генераций в июне, тариф Free» →
   `usage-label` с {month} и {plan} — иначе текст ломается в июле и на платных тарифах.
10. **Тексты демо-переключателя состояний** («Пустое / Пайплайн / Результат / Ошибка»)
    в копидек не включены — это инструмент прототипа, в продукт не переносится.
11. **Третий тон в коде прототипа** помечен `data-tone="firm"`, а в контрактах
    publicReplies — `confident`; копидек закрепляет ключи `*-confident`
    по контракту. Подписи интерфейса не меняются («Уверенный»).
