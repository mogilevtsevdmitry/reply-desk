# MOTION — раскадровка анимации пайплайна генерации

Signature-элемент продукта: «рейка» из четырёх узлов
**Отзыв → Анализ → Тональность → Пакет** с бегущим лучом и последовательным
появлением карточек результата. Всё остальное в продукте анимировано тихо
(120–240 мс); смелость потрачена здесь.

Референс реализации: `screen-generate.html` (состояния «Пайплайн», «Ошибка»,
«Результат» в демо-переключателе). Токены: `--t-fast 120ms`, `--t-base 240ms`,
`--t-slow 480ms`, `--t-stage 1800ms`, `--ease-out cubic-bezier(0.22,0.61,0.21,1)`.

## Привязка к бэкенду

Этапы анимации = статусы SSE генерации. Анимация **событийная, не таймерная**:
узел активируется при получении соответствующего SSE-события, а не по локальному
таймеру. В прототипе тайминги эмулируются `--t-stage` (1.8 с на этап ≈ 8 с всего —
честная длительность реальной генерации).

| Узел | SSE-статус | Подпись под рейкой |
|---|---|---|
| Отзыв | принят в обработку | «Читаю отзыв и выделяю факты…» |
| Анализ | классификация | «Определяю категорию, серьёзность и повторяемость…» |
| Тональность | подбор тона | «Подбираю тон под ваш бренд и правила площадки…» |
| Пакет | сборка результата | «Собираю четыре блока пакета…» |

## Раскадровка (happy path)

**t = 0 мс — переход формы в пайплайн.**
Форма заменяется панелью пайплайна (без cross-fade, мгновенно — пользователь
только что нажал кнопку, контекст очевиден). Вверху панели — первые две строки
отзыва (цитата с левой линейкой): пользователь видит, что обрабатывается именно
его текст.

**Этап n активируется (n = 0…3):**
1. Узел n: кольцо окрашивается в акцент за 240 мс (`--t-base`), внутренняя
   точка получает пульс (scale 1 → 1.5 → 1, 1200 мс, бесконечно, `--ease-in-out`).
2. Подпись под рейкой меняется на текст этапа (aria-live="polite" — статус
   читается скринридером).
3. Луч (`.rail-fill`) дотягивается до узла n за 1800 мс линейно — линейность
   намеренная: луч изображает равномерную работу, а не «прыжки».
4. Узел n−1 гасит пульс и остаётся залитым акцентом (состояние `is-done`).

**t ≈ 7200 мс — все узлы пройдены.**
Последний узел «Пакет» получает `is-done`, панель пайплайна заменяется блоком
результата.

**Появление карточек результата (стаггер).**
Порядок — по убыванию срочности для нервничающего владельца:

| # | Карточка | Задержка | Анимация |
|---|---|---|---|
| 1 | Публичный ответ | 80 мс | opacity 0→1, translateY 14px→0, 480 мс `--ease-out` |
| 2 | Внутренняя задача | 190 мс | то же |
| 3 | Классификация | 300 мс | то же |
| 4 | Возврат клиента | 410 мс | то же |

Интервал 110 мс: волна читается как одно движение, но каждая карточка успевает
зафиксироваться взглядом. Никаких scale/blur — только подъём и проявление.

## Ошибка генерации (status = FAILED)

1. SSE приносит FAILED на этапе k → узел k вместо акцента получает класс
   `is-failed`: кольцо и точка окрашиваются в `--danger`, пульс останавливается,
   луч замирает на достигнутой позиции (не откатывается — честно показываем,
   где остановились).
2. Подпись этапа заменяется блоком ошибки (появление 240 мс):
   - заголовок: **«Не получилось собрать пакет»**
   - текст: «Сервис генерации не ответил вовремя. Лимит не потрачен —
     попробуйте ещё раз, отзыв сохранён.» (формулировка «лимит не потрачен»
     обязательна — см. ADR-002, компенсация лимита при FAILED)
   - кнопка **«Повторить генерацию»** (secondary, не primary — primary
     зарезервирована за стартом AI-действия из формы). Фокус переводится на
     кнопку программно.
3. «Повторить генерацию» (`POST /reviews/:id/retry`, ADR-003) перезапускает
   рейку с нулевого узла: все узлы сбрасываются, луч обнуляется без анимации.

## prefers-reduced-motion: reduce

Статичная версия, функционально эквивалентная:

- луч не анимируется (`transition: none`), заполнение меняется дискретно;
- пульс точки отключён — активный узел отличается цветом кольца и свечением;
- карточки результата появляются все сразу, без стаггера и подъёма;
- смена подписей этапов сохраняется (это контент, не декорация);
- шиммер скелетонов заменяется статичной заливкой `--surface-2`.

## Прочие движения продукта (для полноты)

- Hover/active кнопок и полей: 120 мс, только цвет/бордер. Без transform.
- Переключение табов тона: мгновенная замена текста, подложка таба — 120 мс.
- Тост «Скопировано»: подъём 8px + fade 240 мс, autohide через 2.2 с.
- Шаги онбординга: подъём 8px + fade 240 мс при смене шага.
- Skeleton shimmer: 1400 мс, `--ease-in-out`, бесконечно.
